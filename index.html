Подготовка HTTP-запросов

Все запросы заранее набраны в текстовом редакторе для удобного копирования в PuTTY.
Скриншот 0.4: Запросы в блокноте

Настройка PuTTY

Для отправки запросов используется PuTTY в режиме Raw (прямое TCP-соединение). Адрес: u82358.kubsu-dev.ru, порт 80.
Скриншот 0.5: Конфигурация PuTTY (Raw)

1. Получить главную страницу методом GET в HTTP/1.0

GET / HTTP/1.0

Запрос корневой директории. Сервер возвращает стандартную страницу Apache (так как в корне нет индексного файла).

Результат (скриншот 1.png):

  • Статус: HTTP/1.1 200 OK
  • Заголовки: Server: Apache/2.4.58 (Ubuntu), Content-Length: 10671, Content-Type: text/html
  • Тело: стандартная страница-заглушка Apache2 Ubuntu Default Page.
Скриншот 1: GET / HTTP/1.0

2. Получить внутреннюю страницу методом GET в HTTP/1.1

GET /w2/index.php HTTP/1.1
Host: u82358.kubsu-dev.ru
Connection: close

Запрос index.php из каталога /w2/. Скрипт возвращает статус 404 (задано в коде преподавателем), но выводит данные.

Результат (скриншот 2.png):

  • Статус: HTTP/1.1 404 Not Found
  • Заголовок Content-Type: text/html; charset=UTF-8
  • Тело: Array ( ) Привет, мир!

Статус 404 не мешает анализу содержимого.

Скриншот 2: GET /w2/index.php

3. Определить размер файла file.tar.gz (HEAD)

HEAD /w2/file.tar.gz HTTP/1.1
Host: u82358.kubsu-dev.ru
Connection: close

Метод HEAD возвращает только заголовки. Позволяет узнать размер из поля Content-Length.

Результат (скриншот 3.png):

  • Статус: HTTP/1.1 200 OK
  • Заголовок Content-Length: 11335 байт
  • Тип файла: application/x-gzip
Скриншот 3: HEAD /w2/file.tar.gz

4. Определить медиатип ресурса /image.png

HEAD /w2/image.png HTTP/1.1
Host: u82358.kubsu-dev.ru
Connection: close

Аналогично пункту 3, определяем MIME-тип изображения.

Результат (скриншот 4.png):

  • Статус: HTTP/1.1 200 OK
  • Заголовок Content-Type: image/png
  • Размер: Content-Length: 10506 байт
Скриншот 4: HEAD /w2/image.png

5. Отправить комментарий методом POST

POST /w2/index.php HTTP/1.1
Host: u82358.kubsu-dev.ru
Content-Type: application/x-www-form-urlencoded
Content-Length: 14
Connection: close

comment=Hello

Отправка POST-данных (имитация формы). Скрипт возвращает статус 404, но выводит массив $_POST.

Результат (скриншот 5.png):

  • Статус: HTTP/1.1 404 Not Found
  • Тело: Array ( [comment] => Hello ) Привет, мир!
Скриншот 5: POST /w2/index.php

6. Получить первые 100 байт файла /file.tar.gz

GET /w2/file.tar.gz HTTP/1.1
Host: u82358.kubsu-dev.ru
Range: bytes=0-99
Connection: close

Заголовок Range запрашивает только указанный диапазон байт (первые 100).

Результат (скриншот 6.png):

  • Статус: HTTP/1.1 206 Partial Content
  • Заголовок Content-Range: bytes 0-99/11335
  • Content-Length: 100
  • Тело содержит бинарные данные (первые 100 байт архива)
Скриншот 6: GET /w2/file.tar.gz с Range

7. Определить кодировку ресурса /index.php

HEAD /w2/index.php HTTP/1.1
Host: u82358.kubsu-dev.ru
Connection: close

Запрос HEAD для получения заголовков. Кодировка указана в поле Content-Type.

Результат (скриншот 7.png):

  • Статус: HTTP/1.1 404 Not Found
  • Заголовок Content-Type: text/html; charset=UTF-8 – кодировка UTF-8
Скриншот 7: HEAD /w2/index.php
